### Run-1 anomaly (preserved per the failed-evidence rule)

The first full-tree run (same command, same config, executed under heavy concurrent sibling load)
reported **1002 issues = 991 + 11 `staticcheck` SA5011** hits in `_test.go` files
(`kernel/comment/comment_cov_test.go`, `kernel/httpx/csrf_test.go`, `testkit/auth_options_cov_test.go`)
— sites where `if x == nil { t.Fatal(...) }` guards the dereference, i.e. SA5011 firing only if
staticcheck loses the `t.Fatal` noreturn fact. Two subsequent cache-cleaned runs of the identical
command produced 991 issues with **zero** staticcheck hits, and the committed-config control run
(staticcheck equally enabled) reports 0 issues uncached. A per-analyzer and delta-debug bisect
could not reproduce the 11 hits with any single added analyzer or with the full added set scoped
to an affected package. Conclusion: the 11 SA5011 hits are a **nondeterministic artifact of the
first run** (fact-loss under concurrent load), not a property of the tree. Raw run-1 output is
preserved at `../../artifacts/static-analysis/lint-25-analyzer-run1-anomalous.{txt,json}` and is
not deleted; the authoritative capture is the reproduced 991-issue result. staticcheck is not one
of the CS-23 analyzers; this is flagged so it is not silently absorbed.
