# Invariant ledger

Machine-reviewable table of framework-wide invariants, their enforcement
points, consumer classes, and the regression that guards each one (third
closure audit 2026-07-17). **Rule:** a change that touches an enforcement
point, adds a consumer class, or adds a mutable field to a declaration type
must update the corresponding row (and its regression) in the same PR.

Every public runtime surface has three consumer classes to check: framework-
internal consumers, generated-project consumers (`internal/cli/templates`),
and independently maintained derived projects (stable-v1 source compatibility).

| Invariant | Enforcement | Consumers | Regression |
|---|---|---|---|
| Runtime state is isolated from module-owned declaration memory (no shared mutable storage, however nested) | deep `clone()` at Register and getters (authz, rules, document, notify, workflow); closed scalar model for workflow `Condition.Equals` enforced SYNCHRONOUSLY at `RegisterDefinition` (public API path); `seeds.Bundle.Clone()`; migration FS materialized to immutable byte snapshots at boot (runtime view AND public mirror) | api, worker, migrate, readiness | per-registry alias tests; `TestRegisterDefinitionRejectsMutableEqualsSynchronously`; `TestGatewayRoutingImmuneToRetainedDefinitionMutation`; `TestSeedsI18nAndMigrationContentAreBootCaptured`; `TestIntegrationMaterializedSnapshotMigratesRerunsAndResets` (real goose/PG) |
| Post-boot registration is impossible | every registry sealed by `App.Boot` via `internal/sealer.Authority` (out-of-module boundary; in-repo callers limited to boot by review) | retained `module.Context`, `Booted` live pointers | `TestSealedExtensionModelRejectsEveryPostBootRegistration` (18 mutator classes) |
| The runtime view is the ONLY source of truth: `Booted` is opaque (V2, D-0091) — no informational fields exist, the kernel aggregate pointer is never exposed, and its captured view carries a deep-copied config | opaque struct; `Runtime*` accessors (narrow interfaces for authz/tx); `config.Framework.Clone`; alias-proof template lint (`scripts/lint_templates.sh`, negative fixtures) | StartWorker, Readiness builders, generated api/migrate | `TestKernelReplacementAndMutationCannotAlterRuntimeDependencies`; `TestCapturedKernelConfigIsDeeplyIsolated`; `TestTemplateLintRejectsForbiddenReads` |
| Only boot-produced values can run | `mustBeBooted` panic on accessors; `StartWorker` returns `ErrNotBooted`; no fallback to exported fields | all boot operations | `TestUnbootedBootedFailsLoudly` |
| Declarations are validated at registration or boot, never deferred to first use — on the PUBLIC API path too, with generation-keyed freshness | nil/typed-nil/empty/duplicate/nonpositive rejection everywhere; workflow: synchronous condition validation at RegisterDefinition, RWMutex-guarded registry, mutation bumps a generation that invalidates the last clean Err(), runtime executes only when validation covers current contents; PERSISTED definitions revalidated through the same semantic rules at load | boot + public kernel APIs + DB-loaded definitions | `TestValidationInvalidatedByEveryMutation`, `TestConcurrentRegistrationAndExecutionIsSafe` (-race), `TestIntegrationPersistedVoteDefinitionRejected` (execution + SLA sweep), `TestBootRejectsNilAndEmptyDeclarations`, `TestRegisterRejectsNilAndTypedNilProviders`, ports suite |
| V2 contract consumers keep compiling (positional literals included) | frozen field sets (`app.Hook`, `document.UploadEvent`); new capabilities delivered via new types (`SupervisedHook`) or context (`UploadDeliveryFromContext`); `app.Booted` is intentionally NOT constructible (D-0091) | derived projects | `internal/compat/v2_contract_test.go`; golden consumer gate |
| Hook effects are atomic with the confirming transaction or deduplicable | delivery context carries Tx + retry-stable DeliveryID | document hooks | `TestIntegrationHookEffectsAtomicOrDeduplicatedAcrossRetry` |
| A cancelled bulk aggregate never regains pending items | one-tx Cancel; FOR SHARE aggregate reads in recovery paths; Process entry sweep | bulk workers | `cancel_recovery_test.go` (sequential + commit-window races) |
| Generated products consume only supported accessors | alias-proof template lint (`scripts/lint_templates.sh`, run by the boundary gate) | generated api/migrate/worker | boundary lint gate + negative fixtures + golden consumer runtime |
| Workflow routing is canonical, type-safe, and EXACT (no fmt.Sprint, no Stringer invocation, no float64 precision loss, reload-equivalent) | json.Number ingestion at every decode point + big.Rat comparison; NaN/Inf unrepresentable; callbacks receive deep canonical context copies; CompleteTask serializes output exactly once | workflow runtime + module callbacks | `TestGatewayComparisonIsCanonicalAndTypeSafe`, `TestNumericComparisonExactBoundaries`, `TestIntegrationCallbacksReceiveIsolatedContext`, `TestIntegrationCompleteTaskSerializesOutputOnce` |
| Only boot-produced values operate (V2 opacity, D-0091) | opaque `Booted`; `ErrNotBooted` / fail-loud accessors; no fallback | derived projects | `TestUnbootedBootedFailsLoudly`; decision documented in CHANGELOG + decisions.md |
