# Invariant ledger

This ledger maps each cross-cutting invariant to its representation, executing consumers, and
discriminating regression. A change to an enforcement point, declaration shape, generated consumer,
or persistence contract must update the corresponding row in the same change.

Consumer classes are framework internals, generated products, and independently maintained products
starting at the clean `v1.2.0` baseline.

| Invariant | Enforcement / representation | Consumers | Guarding evidence |
|---|---|---|---|
| Runtime declarations cannot alias module-owned mutable memory | deep clone at registration/getters; closed workflow scalar model; seed clone; boot-materialized migration FS; callback canonical copies | api, worker, migrate, callbacks | registry alias matrices; seed/i18n/migration capture tests; workflow callback isolation |
| Boot is ownership transfer and post-boot mutation is impossible | opaque `app.Booted`; internal runtime view; all registries sealed with internal authority; no exported mirror fields | all processes and retained module contexts | full extension-seal mutator matrix; unbooted fail-loud tests; template access lint |
| Runtime dependencies come only from a validated boot result | narrow `Runtime*` accessors; copied kernel/config state; `ErrNotBooted`; no fallback | StartWorker, readiness, generated api/worker/migrate | kernel replacement/mutation isolation; captured-config clone; generated consumer |
| Declaration validation covers the exact executing generation | synchronous registration checks; generation invalidated only after successful mutation; validation and lookup under one read lock | public workflow API and boot | rejected-mutation no-state-change; every-mutator invalidation; forced concurrent registration/execution under race |
| Workflow persistence is an immutable mirror of registered definitions | one canonical JSON contract; SHA-256 digest; global `(key,version)` identity; atomic sorted sync; stable row IDs; no tenant override | generated migrate, StartIn, Decide, CompleteTask, Delegate, Override, SweepSLA | canonical golden vectors; idempotent/late-conflict sync; corruption, recomputed-digest, missing-registry, and no-side-effect matrices |
| Workflow routing is exact and reload-equivalent | `json.Number` ingestion; rational numeric comparison; immutable condition scalars; canonical callback input; single serialization of output | workflow runtime and module callbacks | numeric-boundary, Stringer, retained-mutation, resolver isolation, and single-marshal tests |
| Only implemented workflow concepts are representable | closed step set: approval, task, auto, gateway, terminal; no vote/ratification/min-approval/self-approval placeholders | module definitions, persistence, runtime | strict parse/validation and public-schema tests |
| Critical process death is observable and terminates the process | one `app.Hook` contract with `Failed`; first failure cancels/drains; reverse bounded stop; worker child supervision | generated api/worker and custom processes | occupied listener, asynchronous failure, sibling cancellation, stop order/timeout tests |
| Upload confirmation effects are authorized by the successful CAS | CAS binds session/document/version/key/checksum/expiry and active document; hook input carries authoritative values, transaction, stable delivery ID | document hooks and scan outbox | wrong identity/expiry/replay zero-hook tests; rollback/retry atomic-or-deduplicated test; retention lock-order races |
| Storage integrity is checksum-bound | `storage.Adapter` requires checksum-aware presign; S3 `Stat` reads canonical metadata and fails on missing metadata; no repair fallback | document upload/confirm and generated storage wiring | memory/S3 checksum mismatch, metadata-only, and generated product tests |
| Cancelled bulk work cannot regain pending items | one-transaction cancel; aggregate row locking in recovery; entry sweep; legal transition CAS | bulk workers and recovery | lifecycle matrix; injected rollback; sequential and commit-window recovery races |
| Retry/recovery remains live under sustained traffic | independent due schedule; bounded error propagation; metrics; fenced leases | outbox relay, jobs, migration backfill | continuous-producer starvation test; injected repeated failure; stale-worker fencing tests |
| Tenant identity is never inferred or substituted | tenant-bound DB handles; exact webhook event/scope match; resource writes require actor-bound context | services, callbacks, fan-outs | RLS census/isolation; zero/mismatch webhook tests; unattributed resource-write rejection |
| Restricted authorization never infers authentication method | authenticators set `Actor.CredentialScheme`; restricted permissions reject zero/unknown; empty `AllowedSchemes` deliberately means method-independent | authz evaluator, document/internal actors, generated auth | credential scheme matrix and static actor-construction scan |
| Privileged state comes from server-side grants | token has opaque grant ID only; canonical principal store resolves membership, assurance, and grant; impersonator/break-glass claims do not exist | JWT auth, audit, workflow override | grant rejection matrix; actor attribution and audit tests; token helper compile surface |
| Mutating HTTP routes declare their request contract | router always enforces request schema or explicit `NoRequestBody`; no rollout flag | built-in and generated routes | route contract matrix; boot failures; generated CRUD consumer |
| Pagination state is authenticated and query-bound | signed opaque envelope includes sort signature; unsigned/flat input rejected; defaults must be positive | list endpoints and generated CRUD | tamper/malformed/sort mismatch/fuzz/order tests |
| Database baseline preserves semantic catalog state | generated clean baseline; full public+migration semantic census; negative mutation oracle | fresh installs, generated migrate, future migrations | catalog equivalence, 29-class discrimination harness, up/down/up, tenant FK/RLS/grant census |
| Release evidence describes work actually executed | one owner per gate; evidence binds source SHA, manifest hash, command hash, exit/status; bundle and attestation revalidate every digest | CI, promotion, release | workflow ownership parser; missing/stale/modified/wrong-SHA evidence matrix |
| Release identity is unique and monotonic | root module; bootstrap `v1.2.0`; abandoned tags reserved; exact tag/commit; proxy non-reuse; future predecessor mandatory | Go module, CLI, templates, release workflows | release contract negative matrix; module/template identity lints; tag-time proxy preflight |
| Generated products use the canonical framework identity and supported APIs | every Go template uses `.FrameworkModule`; hard-coded import lint; generated code consumes runtime accessors only | init/new-module/gen outputs | negative template fixtures; installed golden consumer compile/boot/migrate/serve |
| Optional telemetry never hides a required failure | no-op metrics/tracing only for optional signals; critical worker, scheduler, relay, config, and hook failures still return/report | all processes | no-exporter tests plus critical failure propagation tests |

Independent format versions—configuration schema, events/outbox payloads, workflow definition versions,
audit hash versions, signatures, and performance dataset schema—are not framework-release identities.
They remain explicit evolution boundaries and must fail or migrate according to their own contracts.
